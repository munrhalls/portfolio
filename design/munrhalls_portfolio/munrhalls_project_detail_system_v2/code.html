The challenge

Building a high-performance e-commerce platform demands more than just rendering product cards. The primary architectural challenge for Sang Logium was orchestrating complex client-side state during the checkout flow while maintaining strict server-side validation for pricing and inventory.

We implemented a hybrid rendering approach using Next.js 15, aggressively caching the virtual-filesystem product catalogue on the edge, while keeping the cart and checkout sessions entirely dynamic. This ensured sub-100ms LCP for browsing users without compromising transactional integrity during payment processing.

Architecture

Engineering Decisions

01

Payment Architecture

Implemented Stripe Payment Intents to handle complex 3D Secure authentication flows securely. Built custom webhooks to ensure eventual consistency between the payment gateway and our internal order fulfillment system.

02

Virtual Filesystem

Designed a hierarchical, virtual filesystem structure for product categories. This allowed for infinitely nested sub-categories while maintaining clean, predictable URLs (e.g., /shop/audio/monitors/active).

03

Next.js 15 Implementation

Leveraged React Server Components for heavy data fetching and SEO optimization, reserving Client Components strictly for interactive elements like the mini-cart and checkout forms to reduce JavaScript payloads.
