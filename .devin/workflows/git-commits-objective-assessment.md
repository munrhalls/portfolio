# Git Commits Objective Assessment

Track commits from [date range], start to now. Provide objective, numbers-based professional assessment.

## Required Steps

1. Get git log and stats for all commits
2. For each production code file, read its associated test files
3. Categorize all code: production, test, infrastructure (specs/workflows/conventions)
4. Calculate test-to-production ratio for each feature
5. Assess test coverage: what % of functionality is tested?
6. Report quantitative metrics only:
   - Line counts by category
   - Test-to-production ratios
   - Coverage percentages
   - Quality features (validation, type safety, error handling)
7. Distinguish between tested vs untested code
8. Treat infrastructure as factory, not waste
9. No subjective language ("bullshit", "process theater", etc.)
10. Let numbers speak
